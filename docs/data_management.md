# データ管理およびリポジトリ公開ガイド

本ドキュメントは、「Local Media Library」を GitHub 等の公開リポジトリで安全に公開・共有するための、ファイル構成、データ管理ポリシー、および Git 管理対象と除外対象の区別を整理した資料です。

---

## 1. 起動コマンドと使用データ

本アプリは、利用目的（普段の利用、または初期状態の動作検証）に応じて起動コマンドと保存先ディレクトリが明確に分離される設計になっています。

### 通常利用
* **起動方法**:
  - `start.bat`（ダブルクリック）
  - または `npm start`
* **使用ディレクトリ**: `data/`
* **説明**:
  普段の利用で追加した画像・動画フォルダの情報や、付与したタグ、メモ、お気に入り、作成したコレクションなどはすべてこの `data/` ディレクトリ配下に保存されます。削除すると利用データが失われる可能性があるため、不要な削除は行わないよう注意してください。

### 初回起動・動作検証
* **起動方法**:
  - `start-clean.bat`（ダブルクリック）
  - または `npm run start:fresh`
* **使用ディレクトリ**: `data_test_fresh/`
* **説明**:
  GitHub からクローンした直後の「メディアソースやデータベースが空の初期状態」を検証するための起動モードです。通常の `data/` ディレクトリのデータには影響を与えない設計です。動作検証が終了した後は、必要に応じてフォルダごと削除可能です。削除後に再度起動すると、再度初期状態で自動作成されます。

## 2. Git管理対象にするもの（リポジトリに含めるもの）

以下のファイルおよびフォルダは、本アプリのプログラム本体および公開用ドキュメントであり、個人情報やローカル環境の絶対パスを含まないため、GitHub 等に公開する対象です。

| パス | 用途 | 公開対象とする理由 | 個人情報の有無 |
|---|---|---|---|
| `src/` | バックエンド（Node.js/Express）のソースコード | アプリケーションのサーバーサイドロジック本体のため | なし（個人環境のパス等は含まれません） |
| `public/` | フロントエンド（HTML/CSS/JS）のソースコード | アプリケーションのUIおよび描画ロジック本体のため | なし（汎用的なコードのみで構成されています） |
| `README.md` | リポジトリの概要説明・導入ガイド | 利用者が最初に閲覧する導入および仕様説明資料のため | なし |
| `package.json` | プロジェクト情報および依存パッケージ定義 | 必要なライブラリ構成を定義しインストールを可能にするため | なし |
| `package-lock.json` | 依存パッケージのロック定義 | ライブラリのバージョンを厳密に固定し動作を安定させるため | なし |
| `start.bat` | 通常起動用の Windows バッチファイル | 利用者がダブルクリックで簡単にアプリを起動できるようにするため | なし |
| `start-clean.bat` | 初回検証用の Windows バッチファイル | 導入時のセットアップ挙動を安全にテスト・検証できるようにするため | なし |
| `.gitignore` | Git 管理の除外設定ファイル | 実データやキャッシュなどを誤って追跡対象にしないように制御するため | なし |
| `docs/` | システムドキュメント類（本ガイド等） | データ管理やリポジトリの運用方法を明記し共有するため | なし |

## 3. Git管理対象外にするもの（リポジトリに含めないもの）

以下のファイルおよびフォルダは、ローカルでアプリを実行した際に自動生成されるデータベース、サムネイル画像、キャッシュ、個人設定、または世代バックアップデータです。
これらには利用者の実ファイルパスや、メタデータなどの**個人データ・実データが含まれるため、Git管理対象外（非表示・未追跡）に設定されています**。

| 種別 | パス | 用途・何のために作られるか | 削除の可否と注意点 |
|---|---|---|---|
| **通常データ** | `data/` | 普段利用する際のデータベース、自動生成サムネイル、ログなどが集約される親フォルダ。 | **削除注意**: 削除すると普段使用しているタグやお気に入りなどの情報が消去されます。 |
| **通常データ** | `data/screenshots.db` | 画像のメタデータ、タグ、お気に入り、メモ、コレクション情報を管理する SQLite データベースファイル。 | **削除注意**: 削除するとタグやメモなどの情報がリセットされます。 |
| **通常データ** | `data/thumbnails/` | ギャラリーの表示用に sharp ライブラリが自動生成したサムネイルキャッシュ画像群。 | **削除可**: 削除してもギャラリー表示時に自動で再生成されます。 |
| **通常データ** | `data/logs/` | システム実行ログ（バックアップ実行時など）が出力されるディレクトリ。 | **削除可**: システム動作には影響ありません。 |
| **通常データ** | `data/settings.json` | アプリケーションのシステム設定（画面表示モードなど）が保存されるファイル。 | **削除注意**: 削除すると設定が初期値にリセットされます。 |
| **検証用** | `data_test_fresh/` | `start-clean.bat` 起動時に生成される、初回起動状態検証用の独立したデータディレクトリ。 | **削除可**: 検証完了後はフォルダごと削除可能です。 |
| **バックアップ** | `data_backup_before_changes/` | アップグレードのマイグレーション実行前に作成されるシステム全体の物理退避バックアップ。 | **削除可**: 移行後の動作が正常であることを確認し、過去データへの復元が不要であれば削除可能です。 |
| **バックアップ** | `data/migration_backup/` | 既存データからのマイグレーション時に、世代別に退避されたデータコピー群。 | **削除可**: 古い状態へのロールバックが必要ないことを確認できれば削除可能です。 |
| **一時データ** | `cache/`, `temp/`, `output/` | アプリ動作中に一時的、または明示的な出力で作成されるワークエリア。 | **削除可**: 一時ファイルのため随時削除可能です。 |
| **クリップ出力** | `clips/`, `compressed/` | 動画トリミング機能等により作成された、切り出し動画や圧縮動画の実ファイル出力先。 | **削除注意**: 実ファイルが保存されているため、他の場所へ退避するか、不要になったことを確認して削除してください。 |
| **旧データ** | `videos.json`, `clips.json` | SQLite 移行前の古い動画・クリップ管理用の JSON データ。 | **削除可**: すでに SQLite DB に移行されていれば不要です。 |
| **依存モジュール** | `node_modules/` | `npm install` によりダウンロードされた Node.js のライブラリ群。 | **削除可**: いつでも `npm install` コマンドで再取得可能です。 |
| **環境変数** | `.env`, `.env.local` | ローカルのシステム動作に必要な環境変数定義ファイル。 | **削除注意**: 削除すると一部の設定が初期化される可能性があります。 |

## 4. GitHub公開前チェック手順

新しくリポジトリを初期化して GitHub 等へ公開する際は、以下の手順でコマンドを実行し、個人データや実データが誤って追跡対象に入っていないか最終確認を行ってください。

### 1. Git リポジトリの初期化
```bash
git init
```

### 2. git add 前：未追跡ファイルの確認
```bash
git status
```
* **確認ポイント**: `untracked files`（未追跡ファイル）のリストに、以下のフォルダやファイルが**表示されていないこと**を確認してください。これらが表示されていなければ、`.gitignore` が正常に機能しています。
  - `data/`
  - `data_test_fresh/`
  - `data_backup_before_changes/`
  - `node_modules/`
  - `screenshots.db` などの `.db` ファイル
  - `videos.json` や `clips.json` などのデータファイル
  - 実動画や実画像フォルダ

### 3. 除外（.gitignore の効果）の確認
```bash
git status --ignored
```
* **確認ポイント**: 除外指定されたファイル一覧（`Ignored files`）に、上記の `data/` や `node_modules/` が正しく分類されていることを確認します。

### 4. git add 後：Git管理対象ファイルの確認
ファイルをステージングエリアに追加した後は、`git ls-files` コマンドで、実際に Git の管理対象（追跡対象）となったファイルの一覧を最終確認できます。
```bash
git add .
git ls-files
```
* **確認ポイント**: 表示されるファイルリストの中に、プログラムコードと公式ドキュメント以外の「個人データや実データ」が紛れ込んでいないかを目視で確認します。

## 5. 安全性への配慮と設計方針

本アプリは、個人用ローカルメディアの管理を安全に行うために、以下の設計を採用しています。

* **実ファイルを変更しない設計**:
  登録されたメディアソース内の実ファイルに対して、通常操作からリネーム、移動、削除、上書き加工等を行わない設計です。アプリ側では読み取り専用で参照します。
* **一時的な接続解除への対応**:
  外付けSSDなどが一時的に接続解除され、フォルダが実在しない状態になっても、データベース上の既存レコードにおける missing（未検出）フラグを誤って更新しない設計を採用し、警告ログを出力した上でスキャン処理をスキップします。
* **データのローカル局所化**:
  利用者が登録したメディアフォルダの内部には、アプリ用の一時ファイルやデータベースなどを一切作成しません。アプリ側の生成データは本アプリの `data/` 配下に局所化されます。
