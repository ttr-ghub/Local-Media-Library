# Local Media Library

## 概要

**Local Media Library** は、PC内に保存されている画像や動画などの各種メディアを安全かつ効率的に一元管理できる、ローカル完結型のメディアライブラリ・管理ツールです。

「登録元のオリジナルメディアファイルを変更・削除・上書きしない設計」および「論理登録解除（アーカイブ方式）」を設計の根幹に据え、大切なオリジナルメディアファイルを書き換えたり破損させたりしない高い安全性を実現しています。クラウドと同期するのではなく、利用者のプライバシーと安全なデータ保護を最優先する設計となっています。

---

## 本アプリの設計思想と特徴

本アプリは、プライバシーと安全性を最優先したローカル完結型のメディア管理ツールです。

* **ローカル専用設計**: タグ、お気に入り、メモ、コレクション（メタデータ）などの情報は、すべて外部のクラウドサーバーではなく、利用者のローカルPC内（SQLiteデータベース）にのみ保存されます。
* **外部通信なし**: クラウドへの同期や外部サーバー、SNSへの自動アップロード・画像送信などは行わない設計です。インターネット接続がないスタンドアロン環境でも動作します。
* **オリジナルファイルの完全保護**: 登録されたメディアソースのディレクトリに対して、アプリから書き込み・削除・上書き保存を行わない設計です。アプリ側の操作によってオリジナルの写真や録画データが誤って削除されたり、加工されて上書き保存されたりする事故が根本的に発生しない仕組みを採用しています。
* **格納データの局所化**: メディアフォルダ内にアプリ用の一時キャッシュやデータベースファイルを一切作成しません。アプリの生成データは、すべて本アプリの `data/` ディレクトリ配下にのみ閉じ込めて保存されます。

---

## 主な機能

* **マルチメディアソース管理**: 設定画面から複数の画像・動画ディレクトリを「メディアソース」として動的に追加・有効/無効化でき、サイドバーからディレクトリ単位で簡単に切り替え表示できます。
* **画像ギャラリー**: サムネイル表示、日付グループ化表示、ページネーション、サムネイルサイズ変更（S/M/L/XL）に対応。
* **動画 & クリップ管理**: 登録ディレクトリ内の動画リスト表示、内蔵プレイヤーによる再生、および動画からの特定区間の切り出し（ファストトリミングによる高速非破壊クリップ生成）。
* **安全な論理登録解除（アーカイブ方式）**: メディアソースの登録を解除する際も、SQLite内のメタデータや実ファイル、サムネイルキャッシュなどは削除せず、データベース上でアーカイブ化（論理登録解除）してフラグ管理します。これにより、いつでも以前のお気に入り・タグ・メモを維持したまま復元可能です。
* **整理機能（リンク切れ・物理削除待ちの可視化）**: 
  - 通常ステータス動画の一覧における「ファイル未検出（リンク切れ）」の可視化。
  - 整理タブでの「手動物理削除待ち」および「実ファイル未検出（リンク切れ履歴）」の判別と、リンク切れ履歴の安全なデータベースクリーンアップ機能。
* **メタデータ管理**:
  - コレクション（追加・編集・削除・一括付与）
  - タグ（追加・編集・削除・一括付与）
  - お気に入り（単体トグル・一括設定）
  - メモ書き込み機能（入力確定後にバックグラウンドで自動保存）
* **物理バックアップ**: 利用者が明示的に実行した場合のみ、Robocopyを使用して非破壊的に指定先へバックアップを実行（安全のため、ソース自身やその配下は指定不可にするガード機能を搭載）。

## 技術スタック

* **バックエンド**: Node.js (Express)
* **データベース**: SQLite (Node.js 組み込みの node:sqlite)
* **画像処理**: sharp (高速サムネイル生成)
* **動画処理**: ffmpeg (サムネイル抽出・非破壊トリミング)
* **フロントエンド**: HTML5, Vanilla JavaScript, CSS (CSS変数とBEM設計によるコンポーネント指向)

## セットアップ手順

### 動作環境
* Node.js 24.12.0 以上

1. プロジェクトのルートディレクトリに移動します。

   ```bash

   ```

## 起動方法

### 通常起動
1. プロジェクトルートにある `start.bat` を起動（ダブルクリック）します。
2. 自動的にブラウザが立ち上がり、`http://localhost:3000` が開きます。
3. （コマンドラインから手動で起動する場合は `npm start` を実行してください）

### 初回起動状態の検証・開発用起動 (Clean Verification Mode)
新規利用者がアプリを導入した際の「完全な初期状態（メディアソースやDBが空の状態）」を安全に検証したい場合は、以下のコマンドを使用します。

* **Windows**: プロジェクトルートにある `start-clean.bat` を起動（ダブルクリック）します。
* **その他の環境 / npmスクリプト**: ターミナルで以下のコマンドを実行します。
  ```bash
  npm run start:fresh
  ```

この起動モードでは、通常の `data/` ディレクトリではなく、検証用の独立した `data_test_fresh/` ディレクトリが自動生成されて使用されます。既存のデータや設定には影響を与えないため、何度でも初回セットアップのフローを検証可能です（検証データをリセットしたい場合は `data_test_fresh/` フォルダを削除してください）。

## 初回利用手順 (クイックスタート)

アプリを初めて起動した後は、以下の流れでメディアを追加して使い始めます。

1. **設定画面でフォルダを追加**:
   画面上部またはメニューから「設定」画面を開き、画像または動画が格納されているディレクトリのパスを指定して「登録」を行います。
2. **フォルダの読み込み**:
   登録完了後、画像一覧画面に表示される「画像を読み込む」ボタンをクリックして、指定フォルダ内のファイルの読み込み（スキャン）を実行します。
3. **一覧画面で確認**:
   読み込み処理が完了すると、自動的に画像や動画が日付順にサムネイル化され、ギャラリー画面で自由に管理・整理が可能になります。

## データ保存方針

アプリの動的データは、すべて以下のプロジェクトディレクトリ内にのみ局所化して保存されます。

```
[プロジェクトルート]
  └ data/
     ├ screenshots.db   (SQLite データベース)
     ├ settings.json    (システム設定ファイル)
     ├ thumbnails/      (自動生成された高速表示用サムネイル画像キャッシュ)
     ├ migration_backup/ (マイグレーション時の世代別バックアップフォルダ)
     └ logs/            (バックアップ実行時等のシステムログ)
```

## セキュリティ・安全設計（開発者向けメモ）

本アプリでは、ローカル環境で安全に動作させるため、以下の防御・保護設計を取り入れています。

* **動的パストラバーサル防御 (Security Guards)**:
  画像や動画ファイルをストリーミング・配信するAPIエンドポイントにおいて、要求されたパスが現在データベースに登録され有効化されている「メディアソースのパス」と前方一致するかを動的に判定・チェックするロジックを採用しています。これにより、サーバー上の任意ファイル（システム重要ファイルなど）への不正アクセスを強固に防御しています。
* **一時的な接続解除への対応**:
  外付けSSDなどが一時的に接続解除され、フォルダが実在しない状態になっても、データベース上の既存レコードの `missing` 判定を誤って更新（リンク切れに）しない設計を採用し、警告ログを出力した上でスキャン処理をスキップします。
